## Story
**As a** SDR executando uma cadência,
**I want** clicar em "Ligar via WhatsApp" e falar com o lead direto do navegador,
**so that** eu faça a ligação sem sair da tela de execução, com estados claros de chamada.

## Complexity
**M — Medium** (WebRTC no browser + estados + SSE)

## Scope
**IN:**
- `ActivityWhatsAppCallPanel`, renderizado quando `step.channel==='phone' && step.call_provider==='whatsapp'`.
- Captura de mic (`getUserMedia`), handshake WebRTC via proxy server-side (`/calls` + `/webrtc`).
- Estados de UI: `idle → requesting-mic → ringing → active(timer) → ended` + erros (`mic-denied`, `session-disconnected`, `service-error`).
- Cronômetro inicia **só** na conexão de mídia real (evento `answered`).
- Assinatura do SSE `/events` pra dirigir os estados.
- Seletor de número de destino via `getAllLeadPhones(lead)`.

**OUT:**
- Captura de disposition + avanço (7.6).
- Gravação (7.8).
- Chamada recebida/auto-atender (futuro).

## Dependencies
- 7.1 (serviço + SSE), 7.3 (sessão do SDR pareada), 7.4 (passo).

## Risks
| Risco | Prob. | Impacto | Mitigação |
|-------|-------|---------|-----------|
| Permissão de mic negada | Média | Médio | estado `mic-denied` com instrução; bloqueia ligar |
| SDR sem sessão pareada | Média | Médio | checar `whatsapp_call_sessions.status='connected'` antes de habilitar botão |
| SSE cai no meio da chamada | Baixa | Médio | reconectar SSE; fallback de polling `/history` |
| Cronômetro contar antes de atender | Média | Baixo | timer só no evento `answered` (lição do API4COM: connect ≠ discagem) |

## Acceptance Criteria
1. O painel aparece só para passos `phone` + `call_provider='whatsapp'`.
2. Botão "Ligar via WhatsApp" desabilitado se o SDR não tem sessão `connected` ou o lead não tem número WhatsApp.
3. Ao ligar: pede mic → `ringing` (card "Chamando…") → ao `answered`, vira `active` e **inicia o cronômetro**.
4. O cronômetro reflete só o tempo de conexão real (não conta o ringing).
5. "Desligar" chama `DELETE /calls/{id}`; estado vai pra `ended`.
6. Erros mapeados a estados visíveis (`mic-denied`, `session-disconnected`, `service-error`).
7. Toda comunicação com o microserviço passa pelo proxy server-side (sem API key no browser).
8. Número de destino selecionável quando o lead tem múltiplos (`getAllLeadPhones`).
