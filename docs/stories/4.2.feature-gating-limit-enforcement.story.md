## Status

**Done**

executor: "@dev"

## Complexity Estimate

**M (5 points)** — Guards em múltiplas rotas, componente UpgradePrompt, enforce de limites em actions existentes.

## Risks

- **R1:** Feature gating pode quebrar fluxos existentes para usuários que já usam CRM/Calendar — mitigado porque gating só ativa quando subscription+plan estão carregados
- **R2:** Lead limit enforcement pode bloquear imports parciais — mitigado com contagem prévia antes do import

## Dependencies

- **Story 4.1** — Planos devem estar seedados no banco para que feature flags funcionem

## Story

**As a** platform operator,
**I want** features gated by plan and usage limits enforced,
**so that** users on lower plans cannot access premium features and we can monetize upgrades.

## Scope

**IN:**
- Componente `UpgradePrompt` reutilizável
- Guard de feature para CRM (apenas Pro+)
- Guard de feature para Calendar (apenas Pro+)
- Guard de enrichment level (basic vs lemit vs full)
- Enforce `max_leads` no create-lead e import-leads
- Layout guard: subscription cancelada → redirect para upgrade

**OUT:**
- Feature flags administrativos (toggle manual)
- Granularidade por feature individual além de CRM/Calendar/enrichment
- Rate limiting por API (apenas limites de plano)

## Acceptance Criteria

1. Componente `UpgradePrompt` renderiza: ícone lock, nome da feature bloqueada, plano necessário, botão "Fazer upgrade" → `/settings/billing`
2. Página de integrações CRM mostra `UpgradePrompt` para plano Starter (features.crm === false)
3. Página de integrações Calendar mostra `UpgradePrompt` para plano Starter (features.calendar === false)
4. `createLead()` retorna erro `{ success: false, error: '...', code: 'LEAD_LIMIT_REACHED' }` quando `max_leads` atingido
5. `importLeads()` verifica contagem total antes de iniciar e retorna erro se excederia o limite
6. Subscription com status 'canceled' ou trial expirado bloqueia acesso ao app (redirect para `/upgrade`)
7. `pnpm typecheck`, `pnpm lint` e `pnpm test:run` passam sem erros novos

## Dev Notes

**Reutilizar:** `checkFeature()` de `src/features/billing/services/feature-flags.ts` já existe — usar em Server Components para decidir rendering.

**Pattern para guard em RSC:**
```tsx
// Em page.tsx (Server Component)
const features = await getOrgPlanFeatures();
if (!checkFeature(features, 'crm')) {
  return <UpgradePrompt featureName="Integrações CRM" requiredPlan="Pro" />;
}
return <CrmIntegrationPage />;
```

**Lead count query:**
```sql
SELECT count(*) FROM leads WHERE org_id = $1 AND deleted_at IS NULL
```
