## Complexity Estimate

**M (5 points)** — Serviço de comparação, componente DeltaIndicator, fetch paralelo em múltiplas telas, edge cases de cálculo.

## Risks

- **R1:** Fetch paralelo de 2 períodos pode degradar performance — mitigado: `Promise.all` e queries já otimizadas com filtro de período
- **R2:** Divisão por zero quando período anterior tem valor 0 — mitigado: `safeRate()` já existe no codebase
- **R3:** Métricas onde "menos é melhor" (ex: bounce rate) mostram verde erroneamente — mitigado: flag `invertDelta` no DeltaIndicator

## Dependencies

- **Story 5.1** — Custom Date Range Picker (fornece `from`/`to` e o toggle de comparação)

## Story

**As a** manager,
**I want** to compare current metrics with the previous period,
**so that** I can quickly identify performance trends and take action.

## Scope

**IN:**
- Toggle "Comparar com período anterior" no `DateRangePicker` (criado na 5.1)
- Cálculo automático do período anterior de mesmo tamanho
- Delta % para métricas numéricas em Reports (Overall, Cadence, SDR)
- Delta % para KPIs em Statistics (Activity, Conversion, Team)
- Componente `DeltaIndicator` reutilizável: seta ↑/↓, cor verde/vermelho, valor %
- Tooltip no `DeltaIndicator` mostrando valor absoluto do período anterior
- Serviço de comparação: recebe dados de 2 períodos, retorna deltas
- URL param `compare=true` para persistir estado de comparação

**OUT:**
- Comparação com período customizado (só período anterior automático)
- Gráficos overlay (2 linhas no mesmo chart)
- Comparação YoY (year-over-year)
- Comparação no Dashboard (mantém UX separada)

## Acceptance Criteria

1. Toggle "Comparar" visível no `DateRangePicker`, desativado por default
2. Ao ativar, dados do período anterior são buscados em paralelo
3. `DeltaIndicator` mostra ↑ verde para melhoria, ↓ vermelho para piora
4. Métricas de Reports (Overall): leads, contacted, replied, meetings, qualified mostram delta
5. Métricas de Statistics (Activity KPIs): total atividades, média/dia mostram delta
6. Tooltip no indicator mostra: "Período anterior: X (variação %)"
7. Performance: queries do período anterior não degradam tempo de carregamento > 50%
8. Testes unitários para cálculo de delta (inclusive edge cases: divisão por zero, valores negativos)
9. `pnpm typecheck`, `pnpm lint`, `pnpm test:run` e `pnpm build` passam

## Dev Notes

**Padrão de fetch paralelo:**
```typescript
const [currentData, previousData] = await Promise.all([
  fetchReportData(supabase, orgId, from, to),
  compare ? fetchReportData(supabase, orgId, prevFrom, prevTo) : null,
]);
```

**safeRate() existente:**
```typescript
// src/features/reports/utils/metrics.ts
function safeRate(numerator: number, denominator: number): number {
  return denominator === 0 ? 0 : Math.round((numerator / denominator) * 100);
}
```

**Cálculo do período anterior:**
```typescript
import { subDays, differenceInDays } from 'date-fns';
const days = differenceInDays(to, from);
const prevFrom = subDays(from, days);
const prevTo = subDays(to, days);
```
