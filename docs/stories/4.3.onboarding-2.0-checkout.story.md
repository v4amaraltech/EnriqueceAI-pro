## Complexity Estimate

**L (8 points)** — Expansão de 3 para 6 steps, integração Stripe Checkout no onboarding, Gmail OAuth, persistência de progresso no banco.

## Risks

- **R1:** Redirect Stripe Checkout → onboarding pode perder contexto — mitigado: salvar step no banco antes do redirect
- **R2:** OAuth Gmail pode falhar silenciosamente — mitigado: step com skip, verificação de conexão ao retornar
- **R3:** Onboarding incompleto deixa org em estado inconsistente — mitigado: heurística de redirecionamento já existe (regex domain name), adicionamos onboarding_step

## Dependencies

- **Story 4.1** (Done) — Plans devem existir para step de seleção de plano
- **Story 4.6** (Done) — Invite system com magic links para step de convite

## Story

**As a** new user signing up for the first time,
**I want** a guided onboarding that lets me choose a plan, connect my email, and invite my team,
**so that** I'm fully set up and productive from day one.

## Scope

**IN:**
- Step 1: Nome da empresa (existente, manter lógica atual)
- Step 2: Seleção de plano (3 cards comparativos, incluindo opção "Continuar com trial gratuito")
- Step 3: Checkout Stripe (redirect para Stripe Checkout, retorno com success/cancel)
- Step 4: Conectar Gmail (OAuth via `getGmailAuthUrl()`, com skip)
- Step 5: Convidar equipe (emails + role, com skip, reutilizar `inviteMember`)
- Step 6: Pronto! (checklist do que foi configurado com checkmarks)
- Progress bar atualizado para 6 steps
- Persistência de step atual em `organizations.onboarding_step`
- Retomar onboarding do último step se interrompido

**OUT:**
- Conectar CRM durante onboarding
- Conectar WhatsApp durante onboarding
- Upload de leads CSV durante onboarding
- Feature tour / guided walkthrough
- Onboarding diferenciado por plano

## Acceptance Criteria

1. Onboarding tem 6 steps com progress bar visual (indicadores de step ativo/completo/pendente)
2. Step 1 coleta nome da empresa (mantém comportamento atual via `completeOnboarding`)
3. Step 2 mostra 3 planos com preços e features (reutiliza dados de `fetchPlanComparison`)
4. Step 3 redireciona para Stripe Checkout; ao retornar com `success`, avança para step 4
5. Step 3 permite "Continuar com trial gratuito" (skip checkout, mantém subscription `trialing`)
6. Step 4 permite conectar Gmail via OAuth ou skip (reutiliza `getGmailAuthUrl`)
7. Step 5 permite convidar membros por email com role selector (reutiliza `inviteMember`)
8. Step 6 mostra resumo do que foi configurado com checkmarks verdes
9. Se onboarding interrompido, retoma do último step ao voltar (via `organizations.onboarding_step`)
10. Migration adiciona coluna `onboarding_step` na tabela `organizations`
11. `pnpm typecheck`, `pnpm lint`, `pnpm test:run` e `pnpm build` passam

## Dev Notes

**Onboarding atual (3 steps):**
- `src/app/onboarding/OnboardingWizard.tsx` — client component, state local
- `src/features/auth/actions/complete-onboarding.ts` — salva nome da org
- Trigger: regex `^[a-z0-9.-]+\.[a-z]{2,}$` no nome da org em `layout.tsx`

**Actions reutilizáveis:**
- `createCheckoutSession(planId)` → retorna `{ url }` do Stripe
- `getGmailAuthUrl()` → retorna URL do OAuth consent
- `inviteMember(formData)` → cria magic link + org_member record
- `fetchPlanComparison()` → retorna planos ativos + plano atual

**Fluxo de redirect Stripe:**
```
Onboarding Step 3 → Stripe Checkout → /onboarding?step=3&success=true → Step 4
                                     → /onboarding?step=3&canceled=true → Step 3
```

**Fluxo Gmail OAuth:**
```
Onboarding Step 4 → Gmail consent → /api/auth/callback/gmail → /onboarding?step=4 → Step 5
```

**Progress tracking:**
```
onboarding_step: 0 = empresa, 1 = plano, 2 = checkout, 3 = gmail, 4 = invite, 5 = pronto, NULL = completo
```
