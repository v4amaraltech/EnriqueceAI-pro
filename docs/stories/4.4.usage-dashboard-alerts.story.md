# Story 4.4: Usage Dashboard & Alerts

## Status

**InProgress**

## Executor Assignment

executor: "@dev"
quality_gate: "@architect"
quality_gate_tools: ["lint", "typecheck", "test", "build"]

## Complexity Estimate

**M (5 points)** — Dashboard de uso com barras de progresso, alertas por threshold, gráfico simples de histórico.

## Risks

- **R1:** Contagem de leads pode ser lenta em orgs grandes — mitigado com count() que usa índice
- **R2:** Histórico de AI usage pode ter muitos dias sem dados — mitigado com fill de zeros para dias sem uso

## Dependencies

- **Story 4.1** — Planos devem estar seedados para que limites sejam calculáveis

## Story

**As a** manager,
**I want** a usage dashboard showing my organization's consumption vs plan limits,
**so that** I can monitor usage, anticipate upgrades, and avoid service interruptions.

## Scope

**IN:**
- Seção de uso na página de billing (`/settings/billing`) ou tab separada
- 4 cards de uso: Leads, AI/dia, WhatsApp/mês, Membros
- Barras de progresso com cores por threshold (verde/amarelo/vermelho)
- Notificação in-app quando leads atingem 80% do limite
- Notificação in-app quando AI diário atinge 80%
- Gráfico de uso AI dos últimos 30 dias (line chart simples)
- Apenas managers veem a seção

**OUT:**
- Alertas por email
- Previsão de uso (predictive analytics)
- Detalhamento de uso por SDR individual
- Export de relatório de uso
- Histórico de WhatsApp por dia

## Acceptance Criteria

1. Seção "Consumo" visível na página de billing para managers
2. Card Leads mostra `{current}/{max}` com barra de progresso
3. Card AI mostra `{usedToday}/{maxPerDay}` com barra (ou "Ilimitado" para Enterprise)
4. Card WhatsApp mostra `{usedThisMonth}/{maxPerMonth}` com barra
5. Card Membros mostra `{active}/{included}` com indicador de adicional
6. Barras mudam de cor: verde (<60%), amarelo (60-80%), vermelho (>80%)
7. Notificação enviada para managers quando leads atingem 80% do limite
8. Notificação enviada para managers quando AI diário atinge 80% do limite
9. Gráfico de linha mostrando uso de AI nos últimos 30 dias
10. `pnpm typecheck`, `pnpm lint` e `pnpm test:run` passam sem erros novos

## Dev Notes

**Reutilizar existente:**
- `calculateUsageLimits()` de `feature-flags.ts`
- `isNearLimit()` de `feature-flags.ts`
- `createNotificationsForOrgMembers()` de `notifications/`
- `UsageBar` de `BillingView.tsx` (extrair para componente reutilizável se necessário)

**WhatsApp alert já existe:** `whatsapp-credit.service.ts` já envia alerta de 80% — replicar padrão para AI e leads.

**AI 30 dias query:**
```sql
SELECT usage_date, generation_count
FROM ai_usage
WHERE org_id = $1 AND usage_date >= now() - INTERVAL '30 days'
ORDER BY usage_date ASC
```

**Deduplicate notifications:**
```sql
SELECT 1 FROM notifications
WHERE org_id = $1 AND type = 'usage_limit_alert'
AND metadata->>'channel' = $2
AND created_at::date = CURRENT_DATE
LIMIT 1
```
