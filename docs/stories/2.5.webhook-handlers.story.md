# Story 2.5: Unified Webhook Infrastructure

## Status

**Done**

## Complexity Estimate

**M (Medium)** — Framework de webhooks com validação, idempotência, e logging.

## Risks

- **R1:** Cada provider tem formato diferente de assinatura
- **R2:** Webhooks precisam responder < 200ms para evitar retry storms

## Story

**As a** system,
**I want** a robust webhook infrastructure,
**so that** external events are processed reliably and consistently.

## Acceptance Criteria

1. [x] Framework base de webhook com validação de assinatura
2. [x] Idempotência via event_id (eventos duplicados ignorados)
3. [x] Resposta rápida (< 200ms) — processamento pesado em background via `after()`
4. [x] Logging estruturado para cada evento
5. [x] Dead letter queue para eventos que falharam 3x com retry exponencial
6. [x] Testes unitários (39 testes passando)
