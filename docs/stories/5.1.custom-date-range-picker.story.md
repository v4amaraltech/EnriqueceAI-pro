## Complexity Estimate

**S (3 points)** — Componente UI reutilizável + migração de 3 telas para usar novos URL params.

## Risks

- **R1:** Migrar params de `period` para `from`/`to` pode quebrar bookmarks/links salvos — mitigado: fallback que converte `period` antigo para `from`/`to`
- **R2:** Calendário pode conflitar com tema dark — mitigado: usar primitives do Radix que respeitam CSS vars

## Dependencies

- Nenhuma — esta é a story fundação da Epic 5

## Story

**As a** manager,
**I want** to select custom date ranges for reports and statistics,
**so that** I can analyze data for specific periods beyond the fixed presets.

## Scope

**IN:**
- Componente `DateRangePicker` em `src/shared/components/` baseado em Radix Popover + calendar
- Presets rápidos mantidos (Hoje, 7d, 30d, 90d) como atalhos dentro do picker
- Seleção customizada com calendário (data início e fim)
- Parâmetros de URL: `from` e `to` (ISO date strings) substituindo `period`
- Migrar `StatisticsFilters` para usar o novo componente
- Migrar `ReportsView` period selector para usar o novo componente
- Migrar `PeriodFilter` do Dashboard para usar o novo componente
- Limite máximo de range: 365 dias
- Default: últimos 30 dias quando nenhum período selecionado

**OUT:**
- Comparação entre períodos (Story 5.2)
- Presets relativos complexos ("último trimestre", "YTD")

## Acceptance Criteria

1. Componente `DateRangePicker` renderiza com presets e calendário customizado
2. Seleção de preset atualiza URL params `from`/`to` e refetch dados
3. Seleção customizada permite escolher início e fim com validação (fim >= início, range <= 365d)
4. Statistics, Reports e Dashboard usam o novo componente (presets antigos removidos)
5. URLs com `from`/`to` são shareable (copiar URL preserva filtro)
6. Testes unitários para validação de range e conversão de presets
7. Sem regressão nas 3 telas (dados carregam corretamente com novos params)
8. `pnpm typecheck`, `pnpm lint`, `pnpm test:run` e `pnpm build` passam

## Dev Notes

**Componentes existentes de filtro de período:**
- `src/features/statistics/components/StatisticsFilters.tsx` — Select com presets (today/7d/30d/90d), usa URL param `period`
- `src/features/reports/components/ReportsView.tsx` — Tabs inline (7d/30d/90d), estado local `selectedPeriod`
- `src/features/dashboard/components/PeriodFilter.tsx` — Month selector com setas prev/next
- `src/features/statistics/components/ProspectingSidebarFilters.tsx` — Period preset para sub-pages de prospecting

**Bibliotecas disponíveis:**
- `date-fns` — já instalada, usar para `subDays`, `format`, `differenceInDays`, `startOfDay`, `endOfDay`
- `react-day-picker` — avaliar instalação se necessário, ou implementar com Radix primitives

**Server Actions pattern:**
Todas as actions de fetch usam `createServerSupabaseClient()` e filtram por `created_at` com range. Atualmente recebem um `period` string e calculam `from`/`to` internamente. Inverter: receber `from`/`to` diretamente.

**Relevant Source Tree:**
```

```
