## Status

**InProgress**

## Complexity Estimate

**M (5 points)** — Migration de seed, cron job para expiração, banner de trial no UI.

## Risks

- **R1:** Plans seed em produção pode conflitar se já existirem dados manuais — mitigado com `ON CONFLICT DO NOTHING`
- **R2:** Cron job pode falhar silenciosamente — mitigado com logging e monitoramento
- **R3:** Trial expiry pode bloquear usuários sem aviso — mitigado com banner countdown + notificação 7 dias antes

## Story

**As a** platform operator,
**I want** the 3 plans seeded in the database and trials to expire automatically after 30 days,
**so that** new signups get a real trial experience and we can monetize after the trial period.

## Scope

**IN:**
- Migration seed: 3 planos (Starter, Pro, Enterprise) com todos os limites e features
- Cron endpoint para expirar trials com `current_period_end < now()`
- Banner de trial no app header com countdown de dias restantes
- Notificação in-app 7 dias antes do trial expirar
- Página de "Trial Expirado" bloqueando acesso com CTA upgrade
- Redirect automático para página de upgrade quando subscription cancelada

**OUT:**
- Grace period pós-trial
- Notificação por email de trial expirando
- Planos customizados ou enterprise contact-us
- Planos anuais

## Acceptance Criteria

1. Migration insere 3 planos com limites corretos: Starter (R$149, 1k leads, 50 AI/dia, 500 WA/mês, 4 users, enrichment basic), Pro (R$349, 5k leads, 200 AI/dia, 2.5k WA/mês, 4 users, enrichment lemit, CRM, calendar), Enterprise (R$699, 10k leads, AI ilimitado, 10k WA/mês, 4 users, enrichment full, CRM, calendar)
2. Novo signup cria subscription 'trialing' vinculada ao plano 'starter' automaticamente (trigger existente funciona com seed)
3. Cron endpoint `/api/cron/expire-trials` muda trials com `current_period_end < now()` para status 'canceled'
4. Banner de trial visível no header do app apenas para status 'trialing', mostrando "Seu trial expira em X dias"
5. Notificação in-app enviada para managers quando faltam 7 dias para expirar
6. Subscription com status 'canceled' redireciona para página `/upgrade` com 3 planos e CTA checkout
7. `pnpm typecheck`, `pnpm lint` e `pnpm test:run` passam sem erros novos
