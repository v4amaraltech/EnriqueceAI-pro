## Complexity Estimate

## Risks

- **R1:** Magic link pode não funcionar se Supabase email não estiver configurado — mitigado com fallback para fluxo atual
- **R2:** Expiração de convites pode bloquear membros legítimos — mitigado com opção de reenvio

## Dependencies

- Nenhuma dependência forte — pode ser implementada em paralelo com Wave 1

## Story

**As a** manager,
**I want** a secure invite system without exposed passwords and with invite management,
**so that** I can safely onboard team members and track pending invitations.

## Scope

**IN:**
- Remover exibição de senha temporária do `InviteMemberDialog`
- Convite via Supabase Auth magic link (email de convite padrão)
- Lista de convites pendentes na página de usuários
- Ações: reenviar convite, revogar convite
- Expiração de convites após 7 dias
- Migration: coluna `invited_expires_at` em `organization_members`

**OUT:**
- Email template customizado com branding Flux
- Convite via link compartilhável (share link)
- Bulk invite (CSV de emails)
- Convite via WhatsApp

## Acceptance Criteria

1. `InviteMemberDialog` não mostra senha temporária após enviar convite
2. Dialog mostra mensagem "Convite enviado para {email}" com ícone de sucesso
3. Membro convidado recebe email com link de acesso (Supabase Auth invite)
4. Lista de convites pendentes visível na página `/settings/users` (status 'invited')
5. Manager pode reenviar convite pendente (botão "Reenviar")
6. Manager pode revogar convite pendente (botão "Revogar" → status 'removed')
7. Convites com mais de 7 dias expiram automaticamente (cron ou check on-demand)
8. `pnpm typecheck`, `pnpm lint` e `pnpm test:run` passam sem erros novos
