## Complexity Estimate

## Risks

- **R1:** Tela de execução existente (ActivityQueueView) é completamente substituída
- **R2:** "Modo Execução rápida" é um agrupamento alternativo que precisa de state management
- **R3:** "Leads Aguardando Primeira Ligação" é conceito novo que precisa de query específica

## Story

**As a** SDR,
**I want** uma tela de execução com progresso do dia, objetivo diário, toggle de modo rápido e lista de atividades com filtros,
**so that** eu possa executar atividades de forma organizada e acompanhar meu progresso diário.

## Scope

**IN:**
- Card "Meu Progresso Hoje" (finalizado/pendente)
- Card "Objetivo Diário" com meta
- Tab Execução funcional
- Toggle "Modo Execução rápida"
- Seção "Leads Aguardando Primeira Ligação"
- Lista de atividades com filtros e botão Executar
- Indicador de tempo desde última atividade
- Migration: tabela `daily_activity_goals`

**OUT:**
- Tab Power Dialer (Story 3.17 — placeholder "Em breve")
- Modal de execução split view (Story 3.9)
- VoIP/click-to-call real
- Envio de email/WhatsApp (usa sistema existente)

## Acceptance Criteria

1. Card "Meu Progresso Hoje": "X / Y ATIVIDADES" com indicador verde (finalizado) e cinza (pendente)
2. Card "Objetivo Diário": ícone troféu, "Objetivo diário (N)", texto motivacional, link "Iniciar novas prospecções"
3. Tab "Execução" funcional, tab "Power Dialer" como placeholder "Em breve"
4. Toggle "Modo Execução rápida" agrupa atividades por tipo (todos emails juntos, todas ligações juntas)
5. Seção "Leads Aguardando Primeira Ligação" com cards horizontais (checkbox + avatar + nome + badge tempo)
6. Lista de atividades com filtros: Status, Atividade, Cadência, Passo, Lista de Importação + busca
7. Indicador de tempo desde última atividade em vermelho se > 1h ("HÁ 19 HORAS")
8. Botão "Executar" por atividade (abre modal da Story 3.9 quando disponível)
9. Migration: tabela `daily_activity_goals` com RLS
